# Step 28 - 린팅 후 코드 무결성 검증

린팅 수정 후 코드가 정상적으로 동작하는지 검증한다.

## 작업 흐름

1. **코드 무결성 검증**
   - 컴파일 언어: 빌드/컴파일 실행
   - 인터프리터 언어: 구문 검사 도구 실행

2. **기능 테스트**
   - 기존 테스트 스위트 실행
   - 실패 시 원인 분석 및 기록

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용하여 검증을 수행한다.

**린팅 검증 단계에서 절대로 plan mode를 사용하지 않는다.**

**객관적 검증 기준:**
- 빌드/컴파일 성공 (컴파일 언어)
- 구문 검사 통과 (인터프리터 언어)
- 기존 테스트 통과 (테스트 존재 시)

**검증 실패 시:**
- 실패 원인을 분석하고 기록
- 린팅으로 인한 버그인 경우 코드 수정
- 검증 통과할 때까지 반복

**검증 결과는 청크 단위로 저장한다:**

```
step28_린팅검증_chunk1.md (500줄 이하)
step28_린팅검증_chunk2.md (500줄 이하)
step28_린팅검증_chunk3.md (500줄 이하)
...
```

**작성 규칙**:
- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.
