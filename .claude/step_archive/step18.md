# Step 18 - 코드 리뷰

구현된 코드의 품질을 검토한다.

Class 지향 관점에서 코드 리뷰한다.

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용하여 코드 리뷰를 수행한다.

**코드 리뷰 단계에서 절대로 plan mode를 사용하지 않는다.**

**리뷰 결과는 청크 단위로 저장한다:**

```
step18_코드리뷰_chunk1.md (500줄 이하)
step18_코드리뷰_chunk2.md (500줄 이하)
step18_코드리뷰_chunk3.md (500줄 이하)
...
```

**작성 규칙**:
- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.
