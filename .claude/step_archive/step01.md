# Step 01 - 프로젝트 분석 및 Context 전략 수립

프로젝트 규모를 파악하고 Context 최적화 전략을 수립한다.

## 실행 내용

**프로젝트 규모 파악**: `.claude/hooks/session-start.ps1`에서 자동 출력됨 (sessionStart 시)

Context 최적화 전략을 수립한다:

1. **서브에이전트 전략**: 규모별 개수 결정 (파일 1000개 이상→20개 이상, 100-1000개→10-20개, 100개 미만→5-10개)
2. **파일 읽기 전략**: 500줄 이상은 Grep→offset/limit만, 재읽기 금지
3. **Context 예산**: 메인 10개 이하, 서브에이전트 1-5개 이하

**Context 전략 결과는 청크 단위로 저장한다:**

```
step01_context전략_chunk1.md (500줄 이하)
step01_context전략_chunk2.md (500줄 이하)
step01_context전략_chunk3.md (500줄 이하)
...
```

**작성 규칙**:

- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

## 핵심 원칙 (이후 모든 단계 준수)

1. 500줄 이상 파일 전체 읽기 금지 (Grep→offset/limit만)
2. 같은 파일 재읽기 금지 (기억 의존)
3. 서브에이전트 최대 활용 (복잡한 작업은 분할)

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다.

서브에이전트는 항상 haiku를 사용한다.
