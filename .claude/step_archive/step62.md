# Step 62 - 수정 후 격자 오버레이 디자인 검증

Step 61 수정 후 디자인 시각 검증 완료 후, 격자(Grid) 오버레이를 적용하여 수정된 부분의 정밀한 정렬/간격 검증을 수행한다.

Step 46과 동일한 사전 분석 및 격자 검증 절차를 따르되, 수정 영역에 집중한다.

**이 단계에서 절대로 plan mode를 사용하지 않는다.**

**스크린샷 없이 통과 처리 금지. 반드시 Claude가 직접 스크린샷을 눈으로 확인한다.**

## 격자 오버레이 방식

Step 46의 사전 분석에서 결정된 격자 간격/색상을 재사용한다. 수정으로 인해 디자인 시스템이 변경되었다면 재분석한다.

Playwright의 `page.addStyleTag()`로 런타임 주입한다 (소스 코드 수정 없음).

## 검증 범위

1. **수정된 컴포넌트/페이지** — 변경된 UI 요소가 격자에 맞게 정렬되었는지
2. **수정 영향 범위** — 수정된 요소 주변의 정렬이 격자 기준으로 깨지지 않았는지
3. **전체 페이지 맥락** — 수정된 부분이 전체 격자 구조와 조화되는지

## 검증 절차

1. Step 46에서 결정된 격자 설정 재사용 (변경 시 재분석)
2. 수정 관련 페이지/뷰에 격자 오버레이 적용
3. Playwright 스크립트로 격자 적용 스크린샷 촬영 → `.claude/screenshots/grid/` 저장
4. Claude가 스크린샷을 직접 Read하여 격자 기준 정렬/간격 확인
5. **문제 발견 시:**
   - 어떤 요소가 격자에서 몇 px 벗어났는지 구체적으로 기록
   - 해당 소스 코드(CSS/스타일) 수정
   - 스크립트 재실행 → 스크린샷 재촬영
   - Claude가 다시 직접 확인
   - **통과할 때까지 무한 반복**
6. 모든 항목 통과 확인 후 다음 단계 진행

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다.

서브에이전트는 항상 haiku를 사용한다.
