# Step 61 - 수정 후 디자인 시각 검증

Step 60 구현 완료 후, 수정된 부분과 그 주변 UI가 시각적으로 어색하지 않은지 검증한다.

Step 45와 동일한 검증 태도와 절차를 따르되, 수정 영역에 집중한다.

**이 단계에서 절대로 plan mode를 사용하지 않는다.**

**스크린샷 없이 통과 처리 금지. 반드시 Claude가 직접 스크린샷을 눈으로 확인한다.**

## 🚨 검증 태도 (절대 준수)

Step 45의 검증 태도를 그대로 적용한다:

- **1px 단위까지 의심한다**
- **"거의 맞다"는 "틀렸다"와 같다**
- **빈 공간을 의심한다**
- **일반 사용자의 첫인상으로 본다**
- **스크린샷을 확대해서 본다**
- **비교 검증한다**
- **어색함의 원인을 정확히 특정한다**

## 검증 범위

1. **수정된 컴포넌트/페이지** — 변경된 UI 요소 직접 확인
2. **수정 영향 범위** — 수정된 요소 주변의 레이아웃, 간격, 정렬이 깨지지 않았는지 확인
3. **전체 페이지 맥락** — 수정된 부분이 전체 페이지와 시각적으로 조화되는지 확인

## 검증 절차

1. Playwright로 수정 관련 페이지/뷰의 스크린샷 촬영
2. Claude가 스크린샷을 직접 Read하여 시각적으로 확인
3. **문제 발견 시:**
   - 어떤 요소가 어떻게 어색한지 구체적으로 기록
   - 해당 소스 코드 수정
   - 스크립트 재실행 → 스크린샷 재촬영
   - Claude가 다시 직접 확인
   - **통과할 때까지 무한 반복**
4. 모든 항목 통과 확인 후 다음 단계 진행

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다.

서브에이전트는 항상 haiku를 사용한다.
