# Step 55 - Docker 이미지 빌드 및 검증

프로젝트가 컨테이너화 가능한지 분석하고, Docker 이미지를 빌드하여 정상 동작을 검증한다.

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다.

**Docker 빌드 단계에서 절대로 plan mode를 사용하지 않는다.**

**프로젝트가 Docker와 무관한 경우:**

- 프로젝트 유형에 적합한 배포 환경 검증(패키징, 바이너리 빌드 등)으로 대체

**검증:**

- `.claude/hooks/docker-validator.ps1`에서 자동 검증

**검증 실패 시:**

- 빌드/실행 오류 원인 분석
- Dockerfile 또는 설정 수정
- 검증 통과할 때까지 반복

서브에이전트는 항상 haiku를 사용한다.
