# Step 27 - 코드 린팅

프로젝트의 언어, 기존 린터 설정, 코드 특성을 분석하여 적절한 린팅 전략을 동적으로 결정한다.

포매팅 후 코드 품질을 검사하고 오류를 수정한다.

## 작업 흐름

1. **린팅 실행**
   - 프로젝트 린터 설정에 따라 코드 분석 실행
   - 린팅 오류 및 경고 확인

2. **린팅 이슈 수정**
   - 자동 수정 가능한 항목 우선 처리
   - 수동 수정이 필요한 항목 처리

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용하여 린팅을 수행한다.

**린팅 단계에서 절대로 plan mode를 사용하지 않는다.**

**객관적 검증 기준:**
- 모든 린팅 오류 해결 (린팅 실패 시 린팅 단계 실패)
- 린팅 경고 최소화

**검증:**
- `.claude/hooks/linting-validator.ps1`에서 자동 검증

**검증 실패 시:**
- 린팅 실패 원인을 분석
- 코드를 수정하여 재린팅
- 검증 통과할 때까지 반복

**린팅 결과는 청크 단위로 저장한다:**

```
step27_린팅_chunk1.md (500줄 이하)
step27_린팅_chunk2.md (500줄 이하)
step27_린팅_chunk3.md (500줄 이하)
...
```

**작성 규칙**:
- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.
