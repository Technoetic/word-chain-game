# Step 31 - 에러 핸들링 패턴

프로젝트 언어, 아키텍처, 현재 에러 처리 상태를 분석하여 통일된 에러 처리 전략을 동적으로 수립한다.

## 작업 흐름

1. **현재 에러 처리 패턴 분석**
   - 프로젝트의 에러 처리 패턴 조사
   - 일관성 및 누락 지점 식별

2. **에러 핸들링 개선**
   - 에러 처리 패턴 통일
   - 에러 메시지 표준화
   - 에러 처리 누락 지점 보완

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용하여 에러 핸들링을 개선한다.

**에러 핸들링 단계에서 절대로 plan mode를 사용하지 않는다.**

**에러 핸들링 결과는 청크 단위로 저장한다:**

```
step31_에러핸들링_chunk1.md (500줄 이하)
step31_에러핸들링_chunk2.md (500줄 이하)
step31_에러핸들링_chunk3.md (500줄 이하)
...
```

**작성 규칙**:
- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.
