# Step 05 - UX/UI 조사

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용해 오직 UX/UI 조사만을 수행한다.

## 웹 데이터 수집 방식

**Playwright 필수 사용:**

1. 모든 웹 데이터 수집은 Playwright로만 수행
2. WebFetch/WebSearch 도구 사용 금지
3. 스크린샷과 함께 데이터 수집
4. JavaScript 렌더링된 콘텐츠 캡처
5. UX/UI 요소를 시각적으로 확인하며 조사

**병렬 서브에이전트 실행 시:**

- 각 에이전트가 독립적으로 Playwright 실행
- 스크린샷 파일명 충돌 방지 (에이전트별 고유 prefix)

## 주의사항

- **이 단계에서 절대로 GitHub를 조사하지 않는다.**
- **이 단계에서 절대로 "planning.md"를 생성하지 않는다. 오직 조사만 한다.**
- **이 단계에서 절대로 "design.md"를 생성하지 않는다. 오직 조사만 한다.**
- **이 단계에서 절대로 plan mode를 사용하지 않는다.**

**조사 결과는 청크 단위로 저장한다:**

```
step05_조사결과_chunk1.md (500줄 이하)
step05_조사결과_chunk2.md (500줄 이하)
step05_조사결과_chunk3.md (500줄 이하)
...
```

**작성 규칙**:

- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.
