# Step 23 - 리팩토링

디버깅 완료 후 코드 품질을 개선하기 위해 리팩토링을 수행한다.

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용하여 리팩토링을 수행한다.

**리팩토링 단계에서 절대로 plan mode를 사용하지 않는다.**

**객관적 검증 기준:**
- 모든 테스트 통과 (테스트 실패 시 리팩토링 실패)
- 빌드 성공 (빌드 오류 시 리팩토링 실패)
- 코드 품질 메트릭 유지/개선 (ESLint, Prettier 등 정적 분석 통과)

**검증:**
- `.claude/hooks/refactoring-validator.ps1`에서 자동 검증

**검증 실패 시:**
- 테스트/빌드 실패 원인을 분석
- 코드를 수정하여 재리팩토링
- 검증 통과할 때까지 반복

**리팩토링 결과는 청크 단위로 저장한다:**

```
step23_리팩토링_chunk1.md (500줄 이하)
step23_리팩토링_chunk2.md (500줄 이하)
step23_리팩토링_chunk3.md (500줄 이하)
...
```

**작성 규칙**:
- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.
