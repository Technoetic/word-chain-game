# Step 63 - 수정 회귀 테스트

Step 62 격자 디자인 검증 완료 후, 수정 사항이 기존 기능을 훼손하지 않았는지 검증한다.

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다.

**수정 회귀 테스트 단계에서 절대로 plan mode를 사용하지 않는다.**

## 실행 내용

1. **기존 테스트 전체 재실행**

   - Step 21에서 작성된 테스트 코드 전체 실행
   - Step 40~45 E2E/인터랙션/콘솔 테스트 재실행
   - 실패 항목 기록
2. **수정 영역 집중 검증**

   - 수정된 파일과 연관된 모듈 중심으로 검증
   - 수정 전후 동작 비교
3. **회귀 여부 판정**

   - 수정 전 통과하던 테스트가 실패하면 → 회귀 발생으로 판정
   - 회귀 발생 시 step57.md로 돌아가 재조사

서브에이전트는 항상 haiku를 사용한다.

---

이 지침이 완료되면 모든 수정 작업이 완료된다. 사용자의 추가 수정 요청이 있으면 step57.md부터 시작한다.
