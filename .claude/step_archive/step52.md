# Step 52 - 부하 테스트

프로젝트 특성과 트래픽 패턴을 분석하여 부하 테스트 시나리오와 성능 목표를 동적으로 결정한다.

k6 또는 Artillery를 사용하여 부하 테스트 및 성능 측정을 수행한다.
"웹 앱"이나 "API 서버"가 아니면 프로젝트 유형에 적합한 부하 테스트 도구와 방법을 선택한다.

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다.

**부하 테스트 단계에서 절대로 plan mode를 사용하지 않는다.**

**검증:**

- `.claude/hooks/load-test-validator.ps1`에서 자동 검증

**검증 실패 시:**

- 성능 병목 지점 분석
- 코드 또는 설정 최적화
- 검증 통과할 때까지 반복

**테스트 결과는 청크 단위로 저장한다:**

```
step52_부하테스트_chunk1.md (500줄 이하)
step52_부하테스트_chunk2.md (500줄 이하)
step52_부하테스트_chunk3.md (500줄 이하)
...
```

**작성 규칙**:

- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.
