# Step 36 - 최적화 검증

최적화 구현 결과를 합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용하여 검증한다.

**검증 단계에서 절대로 plan mode를 사용하지 않는다.**

**검증 결과는 청크 단위로 저장한다:**

```
step36_최적화검증_chunk1.md (500줄 이하)
step36_최적화검증_chunk2.md (500줄 이하)
step36_최적화검증_chunk3.md (500줄 이하)
...
```

**작성 규칙**:
- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.

---

이 지침을 완료한 즉시 자동으로 step37.md를 읽고 수행한다. 사용자 확인을 기다리지 않는다.
