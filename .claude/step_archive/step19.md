# Step 19 - 테스트 파일 매핑

step18_코드리뷰_chunk*.md 기반으로 테스트 파일 매핑 및 서브에이전트 배정 계획을 수립한다.

## 실행 내용

1. **대상 파일 목록**: Glob으로 구현 파일 검색, step18_코드리뷰_chunk*.md에서 우선순위 추출
2. **복잡도 분석**: 테스트 난이도, 외부 의존성(API/DB/파일), Mock 필요성 판단
3. **서브에이전트 배정**: 파일당 1개 배정, 규모별 개수 (소규모 10-20개, 중규모 20-50개, 대규모 50개 이상)
4. **읽기 전략**: 담당 파일 1개만 Read, 대용량은 Grep→offset/limit
5. **테스트 지침**: 프레임워크, 커버리지 목표, 네이밍, Mock 전략
6. **Context 격리**: 독립 context, 파일 1개만 읽고 테스트 1개만 작성

**매핑 결과는 청크 단위로 저장한다:**

```
step19_테스트매핑_chunk1.md (500줄 이하)
step19_테스트매핑_chunk2.md (500줄 이하)
step19_테스트매핑_chunk3.md (500줄 이하)
...
```

**작성 규칙**:

- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다 (20-50개 이상).

서브에이전트는 항상 haiku를 사용한다.

**테스트 매핑 단계에서 절대로 plan mode를 사용하지 않는다.**
